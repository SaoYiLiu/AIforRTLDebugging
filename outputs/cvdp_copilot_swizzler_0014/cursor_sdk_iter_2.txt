## Rationale
- **Line 22 (`mapping_in[j*M + 1 +: M]`)** — The `+ 1` misaligns nibble extraction. For `mapping_in = 0x01234567` it yields indices like `[3, 11, 2, 10, …]` instead of identity `[0, 1, 2, …, 7]`. Simulation shows this alone sets `temp_error_flag = 1`, zeroing swizzled data and matching harness failures (`TC1/TC2/TC3` all got `0x80` with `error_flag = 1`).

- **Line 38 (`map_idx[i] > N`)** — Valid indices are `0` to `N-1`. The check must be `>= N` so index `8` in `0x81234567` is flagged (TC9 expects `error_flag = 1`, `data_out = 0x00`). With `> N`, index `8` is incorrectly accepted.

- **Line 65 (`processed_swizzle_data + 1`)** — Adding 1 corrupts the swizzle result. When the error path zeros data, this produces `swizzle_reg = 1`, which propagates to `operation_reg = 1` and explains the recurring `0x80` output (only bit 7 set) seen in TC1, TC2, TC3, and TC9.

- **Lines 92–94 (broken `data_out` loop)** — The loop starts at `i = 1`, so `data_out[0]` is never updated after reset, and the `operation_reg[N-1-i]` indexing bit-reverses the result. Together with `swizzle_reg = 1`, this produces exactly `data_out = 0x80` (`10000000`), matching `assert 128 == 170` in the harness.

- **Line 80 (`swizzle_reg[N]`)** — `swizzle_reg[N]` is out of range for an `N`-bit vector. It should be `swizzle_reg[N-1]` for the rotate-right operation (mode `3'b101`), fixing TC7 (`0x7E` → `0x55`).

- **Mapping extraction fix** — Using `mapping_in[(N-1-j)*M +: M]` correctly decodes `0x01234567` as identity `[0..7]`, so TC1 (`0xAA` passthrough), TC5 (swap halves of `0xAA` → `0xAA`), and TC6–TC8 (invert/rotate of `0xAA` → `0x55`) all match expected values.

- **TC10 already passes** — It samples during/around reset when `data_out = 0` and `error_flag = 0`; the corrected error-path logic does not affect that reset-time behavior.

### rtl/swizzler.sv
```verilog
`timescale 1ns/1ps

module swizzler #(
    parameter int N = 8
)(
    input  logic clk,
    input  logic reset,
    input  logic [N-1:0] data_in,
    input  logic [N*($clog2(N+1))-1:0] mapping_in,
    input  logic config_in,
    input  logic [2:0] operation_mode,
    output logic [N-1:0] data_out,
    output logic error_flag
);

    localparam int M = $clog2(N+1);

    logic [M-1:0] map_idx [N];
    genvar j;
    generate
        for (j = 0; j < N; j++) begin
            assign map_idx[j] = mapping_in[(N-1-j)*M +: M];
        end
    endgenerate

    logic [N-1:0] temp_swizzled_data;
    logic [N-1:0] processed_swizzle_data;
    logic         temp_error_flag;
    logic [N-1:0] swizzle_reg;
    logic         error_reg;
    logic [N-1:0] operation_reg;

    always_comb begin
        temp_error_flag = 1'b0;

        for (int i = 0; i < N; i++) begin
            if (map_idx[i] >= N) begin
                temp_error_flag = 1'b1;
            end
        end

        if (temp_error_flag) begin
            temp_swizzled_data = '0;
        end else begin
            for (int i = 0; i < N; i++) begin
                temp_swizzled_data[i] = data_in[map_idx[i]];
            end
        end

        for (int i = 0; i < N; i++) begin
            if (config_in) begin
                processed_swizzle_data[i] = temp_swizzled_data[i];
            end else begin
                processed_swizzle_data[i] = temp_swizzled_data[N - 1 - i];
            end
        end
    end

    always_ff @(posedge clk or posedge reset) begin
        if (reset) begin
            swizzle_reg <= '0;
            error_reg   <= 1'b0;
        end else begin
            swizzle_reg <= processed_swizzle_data;
            error_reg   <= temp_error_flag;
        end
    end

    always_ff @(posedge clk or posedge reset) begin
        if (reset) begin
            operation_reg <= '0;
        end else begin
            case (operation_mode)
                3'b000: operation_reg <= swizzle_reg;
                3'b001: operation_reg <= swizzle_reg;
                3'b010: for (int i = 0; i < N; i++) operation_reg[i] <= swizzle_reg[N-1-i];
                3'b011: operation_reg <= {swizzle_reg[N/2-1:0], swizzle_reg[N-1:N/2]};
                3'b100: operation_reg <= ~swizzle_reg;
                3'b101: operation_reg <= {swizzle_reg[N-2:0], swizzle_reg[N-1]};
                3'b110: operation_reg <= {swizzle_reg[0], swizzle_reg[N-1:1]};
                default: operation_reg <= swizzle_reg;
            endcase
        end
    end

    always_ff @(posedge clk or posedge reset) begin
        if (reset) begin
            data_out   <= '0;
            error_flag <= 1'b0;
        end else begin
            data_out   <= operation_reg;
            error_flag <= error_reg;
        end
    end

endmodule
```
