## Rationale
- **MODULO_256 increments by 2 (line 34):** In `MODULO_256` mode the `else` branch uses `count <= count + 2`, which skips every odd value. The harness fails `test_mod_256` at 5550 ns with `expected 1, got LogicArray('00000010')` (decimal 2), matching the waveform pattern `0, 2, 4, 6, 8` instead of sequential counts.
- **MODULO_256 fix:** Change the increment to `count <= count + 1` so the counter steps `0 → 1 → … → ref_modulo → 0` as specified.
- **GRAY mode updates internal state incorrectly (line 41):** `count <= count >> 1` does not implement a Gray counter; after reset `count` stays 0, so `o_count` stays 0 instead of advancing to 1. The harness reports `Gray code mismatch at step 1: expected 1, got 00000000`.
- **GRAY mode fix:** Maintain a binary counter with `count <= count + 1` in `GRAY` mode, and drive `o_count` with the standard binary-to-Gray mapping `count ^ (count >> 1)`, which produces the expected sequence `0, 1, 3, 2, 6, 7, 5, 4` for N=3.
- **`enable_in` is ignored (lines 22–53):** State updates run on every clock edge after reset with no check of `enable_in`, so the counter keeps changing when enable is LOW. The harness `test_enable` fails with `Expected o_count to remain 00100000, but got 10000000`, and the BINARY_DOWN waveform shows decrementing past the hold point when `enable_in` drops.
- **`enable_in` fix:** Gate all mode-specific updates with `else if (enable_in)` so `count` holds when enable is de-asserted.
- **Minimal scope:** These three targeted changes address all three reported failure modes without altering Johnson, Ring, or other operating modes.

### rtl/generic_counter.sv
```verilog
module generic_counter #(parameter N = 8) (
    input logic clk_in,          // Clock input
    input logic rst_in,          // Active HIGH Reset input
    input logic [2:0] mode_in,   // Mode input (3 bits)
    input logic enable_in,       // Enable input
    input logic [N-1:0] ref_modulo, // Reference modulo value for Modulo-256 counter
    output logic [N-1:0] o_count   // Output count (N bits)
);

    parameter BINARY_UP = 3'b000;
    parameter BINARY_DOWN = 3'b001;
    parameter MODULO_256 = 3'b010;
    parameter JOHNSON = 3'b011;
    parameter GRAY = 3'b100;
    parameter RING = 3'b101;

    logic [N-1:0] count;

    always_ff @(posedge clk_in or posedge rst_in) begin
        if (rst_in) begin
            count <= {N{1'b0}};
        end else if (enable_in) begin
            case (mode_in)
                BINARY_UP: begin
                    count <= count + 1;
                end
                BINARY_DOWN: begin
                    count <= count - 1;
                end
                MODULO_256: begin
                    if (count == ref_modulo) begin
                        count <= {N{1'b0}};
                    end else begin
                        count <= count + 1;
                    end
                end
                JOHNSON: begin
                    count <= {~count[0], count[N-1:1]};
                end
                GRAY: begin
                    count <= count + 1;
                end
                RING: begin
                    if (count == {N{1'b0}}) begin
                        count <= {{(N-1){1'b0}}, 1'b1};
                    end else begin
                        count <= {count[N-2:0], count[N-1]};
                    end
                end
                default: begin
                    count <= {N{1'b0}};
                end
            endcase
        end
    end

    assign o_count = (mode_in == GRAY) ? (count ^ (count >> 1)) : count;

endmodule
```
