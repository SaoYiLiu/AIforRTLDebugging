## Rationale
- **Cycle 5 passes but cycle 7 fails** (`42243 × 58071`: expected `173882245`, got `-249701452`), so the left-shift and `temp_products` fixes are correct for isolated ops but **back-to-back pipelined multiplies still break**.
- **Lines 40–46 and 63–88 use `valid_reg` as one-hot state, not a shift register.** The spec requires a **shift register** to track multiple in-flight operations; the old `valid_reg[0]` blocking also limits throughput to one input every two cycles.
- **Lines 64, 75, 86, and 97–100 aggressively clear state** (`valid_reg` bits and `result <= 0`), which corrupts overlapping pipeline transactions when the cocotb test queues many random multiplies (5th result OK, 7th wrong).
- **Stage 3 partial-sum reads and stage 2 Booth writes can occur in the same cycle** (`valid_pipe[3]` and `valid_pipe[4]` both set). Nonblocking assignments correctly use the **previous** `partial_products` for summation while Booth updates for the next op — but only if control is a proper shift register, not cleared one-hot flags.
- **Fix:** replace `valid_reg` with `valid_pipe <= valid_pipe >> 1` plus `valid_pipe[4] <= 1'b1` on `start`; drive stages with `valid_pipe[4:1]` (Booth → partial → s3/s4 → output); latch inputs on every `start`; keep `<<` shifts and `temp_products1/2`; stop zeroing `result` when idle (only deassert `done`).

```verilog
module pipelined_modified_booth_multiplier (
    input clk,
    input rst,
    input start,
    input signed [15:0] X,
    input signed [15:0] Y,
    output reg signed  [31:0] result,
    output reg done
);

    reg signed [31:0] partial_products [0:7];
    reg signed [15:0] X_reg, Y_reg;
    reg [4:0] valid_pipe;

    integer i;

    // Registers for pipelining the addition stages
    reg signed [31:0] s1, s2, s3, s4;
    reg signed [31:0] temp_products1, temp_products2;

    always @(posedge clk or posedge rst) 
    begin
        if (rst) 
        begin
            X_reg <= 16'd0;
            Y_reg <= 16'd0;
            valid_pipe <= 5'd0;
            done <= 1'b0;
            for (i = 0; i < 8; i = i + 1) 
            begin
                partial_products[i] <= 32'd0;
            end
            s1 <= 0; s2 <= 0; s3 <= 0; s4 <= 0;
            temp_products1 <= 0; temp_products2 <= 0;
            result <= 32'd0;
        end 
        else 
        begin
            // Stage 1: Input registering
            if (start) begin
                X_reg <= X;
                Y_reg <= Y;
            end

            // Stage 2: Booth encoding and partial product generation
            if (valid_pipe[4]) begin
                for (i = 0; i < 8; i = i + 1) begin
                    case ({Y_reg[2*i+1], Y_reg[2*i], (i == 0) ? 1'b0 : Y_reg[2*i-1]})
                        3'b000, 3'b111: partial_products[i] <= 32'd0;
                        3'b001, 3'b010: partial_products[i] <= {{16{X_reg[15]}}, X_reg} << (2*i);
                        3'b011: partial_products[i] <= {{16{X_reg[15]}}, X_reg} << (2*i + 1);
                        3'b100: partial_products[i] <= -({{16{X_reg[15]}}, X_reg} << (2*i + 1));
                        3'b101, 3'b110: partial_products[i] <= -({{16{X_reg[15]}}, X_reg} << (2*i));
                        default: partial_products[i] <= 32'd0;
                    endcase
                end
            end

            // Stage 3: Partial product reduction
            if (valid_pipe[3]) begin 
                s1 <= partial_products[0] + partial_products[1] + partial_products[2];
                s2 <= partial_products[3] + partial_products[4] + partial_products[5];
                temp_products1 <= partial_products[6];
                temp_products2 <= partial_products[7];
            end 

            // Stage 4: Final summation
            if (valid_pipe[2]) begin 
                s3 <= s1 + s2;
                s4 <= temp_products1 + temp_products2;
            end

            // Stage 5: Output result
            if (valid_pipe[1]) begin
                result <= s3 + s4;
                done <= 1'b1;
            end 
            else begin
                done <= 1'b0; 
            end 

            // Shift-register valid propagation through the pipeline
            valid_pipe <= valid_pipe >> 1;
            if (start)
                valid_pipe[4] <= 1'b1;
        end
    end
        
endmodule
```
