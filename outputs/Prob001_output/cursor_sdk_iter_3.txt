## Transport
rest

## Rationale
- **Suspected bug (L010–L017):** The merged block compares the correct next value `{a, a_tem[6:0]}` at **L016**, but **`match` is still registered** on `posedge clk`. That makes `match` assert one cycle late and de-assert one cycle late relative to the reference.
- **Evidence:** `match_ref` asserts at **195 ps** (same time as `a` → 1), while `match_dut` stays **0** through the failure at **200 ps**. The reference reacts combinationally when the incoming bit completes the pattern; the DUT waits for the clock edge.
- **Two mismatches explained:** At **195 ps**, ref=1 and dut=0 (late assert). At **205 ps**, ref=0 (pattern broken once `a_tem` shifts) but dut would still be 1 from the registered update at **200 ps** (late de-assert).
- **Waveform causal trace:** The failure is tied to `a` transitioning to 1 at **195 ps** — the final bit of `8'b0111_0001` — not to a delayed registered response.
- **Fix:** Keep `a_tem` as a clocked shift register, but drive **`match` combinationally** from the effective 8-bit window `{a, a_tem[6:0]}`, gated by `rst_n`.

## FixedTopModule
```verilog
module TopModule(
    input clk,
    input rst_n,
    input a,
    output match
);

    reg [7:0] a_tem;

    always @(posedge clk or negedge rst_n)
        if (!rst_n)
            a_tem <= 8'b0;
        else
            a_tem <= {a, a_tem[6:0]};

    assign match = rst_n && ({a, a_tem[6:0]} == 8'b0111_0001);

endmodule
```
