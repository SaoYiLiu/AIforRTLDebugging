## Rationale
- **Suspected bug (L009):** The next-state expression `Q <= {Q[0], Q[3:1]}` is a plain right rotate. A Johnson counter must shift right while injecting the **inverted** LSB into the MSB, not the LSB itself.
- **Why it fails from reset:** From `0000`, `{Q[0], Q[3:1]}` evaluates to `{0, 000} = 0000`, so the counter never leaves the reset state.
- **Waveform evidence:** At `t=5` ps both `Q_dut` and `Q_ref` are `0` after async reset. At `t=95` ps (posedge clock), `Q_ref` advances to `1000` but `Q_dut` remains `0`, triggering the first mismatch at `t=100` ps (`352/421` samples wrong).
- **Expected sequence:** `0000 → 1000 → 1100 → 1110 → 1111 → 0111 → 0011 → 0001 → 0000`. A rotate without inversion cannot produce this walking-ones then walking-zeros pattern.
- **Correct Johnson logic:** `{~Q[0], Q[3:1]}` shifts `Q[3:1]` right and feeds `~Q[0]` into bit 3. While LSB is 0, MSB gets 1 (walking ones); once LSB is 1, MSB gets 0 (walking zeros).
- **Fix applied:** Change L009 from `{Q[0], Q[3:1]}` to `{~Q[0], Q[3:1]}`. Reset logic on L007–L008 is already correct.

## FixedTopModule
```verilog
module TopModule(
   input                clk ,
   input                rst_n,
 
   output reg [3:0]     Q  
);
    always@(posedge clk or negedge rst_n)begin
        if(!rst_n) Q <= 'd0;
        else Q <= {~Q[0], Q[3 : 1]};
    end
endmodule
```
